# 個人情報・公開データ方針 v2026.05.06-alpha.5.4

公開デモ版で外部に出る情報の安全方針です。本リリースパックの全ドキュメントとサンプル出力に適用します。

---

## 1. 公開サンプルは架空データのみ

- 公開サンプル（`out/sample_*_public.md`）はすべて**公開デモ用の架空サンプル**です
- 実利用者・実職員・実事業所の情報は含まれません
- 事業所コードは `DEMO-0004 / DEMO-0005 / DEMO-0006 / DEMO-0007` の架空コードを使用します

## 2. 公開出力に含めない情報（PII禁止項目）

| 項目 | 公開出力 |
|---|---|
| 実利用者氏名・かな | 含めない |
| 被保険者番号（10桁） | 含めない |
| 実利用者の住所・電話・生年月日 | 含めない |
| 実利用者の家族情報・続柄 | 含めない |
| 利用者の医療機関名・主治医名 | 含めない |
| 利用者の具体的病名・診断名 | 含めない |
| 実職員氏名 | 含めない |
| 実職員の住所・電話・生年月日 | 含めない |
| 実職員の給与額・賞与額 | 含めない |
| 資格証番号・免許番号 | 含めない |
| 入職日・退職日 | 含めない |
| 実事業所コード（10桁の指定事業所番号） | 含めない |
| 実社名（事業者名・施設名） | 含めない |
| メールアドレス | 含めない |

## 3. raw データの公開禁止

| データ種別 | 公開出力 | 理由 |
|---|---|---|
| 公開デモ用の架空職員サマリ（生ファイル） | 出さない | 個別職員の集合体であるため |
| 公開デモ用の架空証跡データ（生ファイル） | 出さない | 内部fact pathが含まれるため |
| 公開デモ用の架空利用者集計（生ファイル）alpha.5.4追加 | 出さない | 内部fact pathが含まれるため |
| evidence JSON（PDF抽出結果） | 出さない | 集計値を含むため、社外公開時には集計レベルでも追加レビュー必要 |
| 内部レポート（DEMO接尾辞なし版） | 出さない | 実事業所データの可能性 |
| 加算マスタファイル本体 | 出さない | 法令解釈・社内ナレッジ |
| 内部資料・社内regulatory資料への参照 | 出さない | 内部資料パス |

## 4. 職員サマリ表示の方針

公開出力では**集計値のみ**を表示します。

- 表示する: 件数（看護職員数・介護職員数等）・常勤換算合計・比率（介護福祉士比率等）
- 表示しない: `staff_id`・`display_label`・個別の資格詳細・個人別シフト

職員サマリは公開デモ用の**架空職員サマリ**から生成され、本番tenantのデータとは無関係です。

## 5. 利用者集計表示の方針（alpha.5.4 追加）

公開出力では**集計値のみ**を表示します。

表示する:
- 利用者総数・要介護度分布（分類ごとの件数）
- 要介護3以上の件数・比率
- 要介護4以上の件数・比率
- 重度者の件数・比率
- 認知症日常生活自立度分布
- 看取り期対応関連の件数
- 医療依存度のある利用者の件数
- 退院退所支援関連の件数
- 緊急時対応関連の件数

表示しない:
- 個別利用者の氏名・かな
- 被保険者番号・住所・電話・生年月日
- 家族情報・続柄
- 医療機関名・主治医名
- 具体的病名・診断名

利用者集計は公開デモ用の**架空集計**から生成され、本番tenantのデータとは無関係です。

## 6. 内部表現の publicへの非露出

- 内部資料パス（社内マスタ参照等）は public に出さない
- 内部用語（架空データ用の社内呼称等）は public に出さない
- 内部資料への参照（社内regulatory資料等）を public に書かない
- raw user_summary.json / raw staff.json / raw tenant_status.json への内部パスを公開向け資料に不用意に出さない
- 代わりに「公開デモ用の架空サンプル」「公開デモ用の架空職員サマリ」「公開デモ用の架空証跡データ」「公開デモ用の架空利用者集計」と表現する

## 7. 内部fact pathの非露出

要件DSL の内部fact pathは公開向け日本語labelに変換してから表示します。

- 例: `tenant_status.monthly_meeting_record.status` → 「月次会議議事録」
- 例: `staff_summary.helper_kaigo_fukushishi_ratio` → 「ヘルパーの介護福祉士比率（職員データ集計）」
- 例: `user_summary.severe_user_ratio` → 「重度者の利用者割合（利用者集計）」

`config/evidence_labels.json` に label / 推奨確認資料 / 優先度 / 次アクション を辞書として登録しており、公開出力時に変換されます。

## 8. 危険表現の禁止

公開出力では以下のような断定的・誤解を招く表現を**使わない**:

- 算定可否を法的に保証するかのような断定文言
- 算定が確実視される文言
- PDF未検出を未算定と断定する文言
- 自治体確認の必要性を否定する文言
- 「100%」「絶対」など完全保証を意味する文言
- 「公式コード表と完全照合済み」のような未達成事項を誇張する表現
- 「本番SaaSとして即導入可能」のような未対応事項を誇張する表現

否定形で**使う**のは可:

- 「算定可否を**保証するものではありません**」
- 「未検出は**未算定を意味しません**」
- 「対象外加算が改善候補・収益機会に**混入しない**」

## 9. public sampleの安全スキャン項目

公開出力を生成・更新する前に必ず以下を通します。

- 禁止語チェック（実社名・実事業所コード・実職員名・実利用者名・内部表現）
- PIIチェック（被保険者番号・電話・メール・生年月日・住所・給与）
- 危険表現チェック（保証・必ず・絶対・100%）
- raw staff露出チェック（`staff_id` の値露出・個別職員リスト）
- raw tenant_status露出チェック（個別fact値の生露出）
- raw user_summary露出チェック（個別利用者ID・名前・住所・生年月日等）
- 内部fact path生露出チェック（公開向け日本語labelに変換済か）
- not_applicable 誤表示チェック（対象外加算が改善候補・収益機会に混ざっていない）
- pattern_based_unverified の誤 clear チェック
- demo_aggregate_unverified の誤 clear チェック

すべて pass してから公開してください。
