# CareLinker 加算チェッカー 商品概要

**バージョン**: v2026.05.06-alpha.5.4（限定外販MVP / 公開デモ版）

---

## 誰向けか

- 介護事業所（訪問介護・居宅介護支援・通所介護・訪問看護）の**経営者・管理者**
- 多施設展開している介護法人の**本部・経理担当**
- 介護**コンサルタント・社労士・行政書士**

---

## 主な機能

### 1. PDF加算検出

レセプトPDFから現在算定中と推定される加算を自動抽出します。

- 4サービス対応（通所介護・訪問介護・居宅介護支援・訪問看護介護保険版）
- 推定利用者数・要介護度分布・抽出信頼度を併せて提示
- 個人情報（被保険者番号・氏名・住所・電話）は意図的に保存しない

**注意**: PDF検出は「算定中の推定」です。要件充足を保証するものではありません。

### 2. 要件ロジック評価（DSL）

公式根拠確認済みの要件についてAND/OR/ネスト評価で機械推定します。

評価結果ステータス:
- `clear` / `not_clear` / `partially_clear`
- `blocked_by_missing_evidence`（不足証跡あり）
- `blocked_by_unverified_mapping`（サービスコード照合未完了で保留）
- `not_evaluated_source_required`（根拠未確認で評価せず）
- `not_evaluated_logic_unchecked`（ロジック未確認で評価せず）
- `not_applicable`（当サービス対象外）

### 3. tenant_status（事業所届出状況）連携

事業所側の届出状況・運用書類整備状況等を要件評価に供給します。

- 公開デモでは `DEMO-0004` 等の架空事業所コードを使用
- 実事業所のtenant_statusは公開出力に含めません

### 4. staff_summary（職員データ）連携

公開デモ用の架空職員サマリから集計値を要件評価に投入します。

- 表示するもの: サ責資格保有者数・上乗せ配置常勤換算・ヘルパー介護福祉士比率・看護職員数・常勤看護職員数等
- 表示しないもの: 個別の氏名・staff_id・資格詳細・個人別シフト

### 5. user_summary（利用者集計）連携 — alpha.5.4 追加

公開デモ用の架空利用者集計から、**集計値のみ**を要件確認補助として供給します。

- 表示するもの: 利用者総数・要介護度分布・重度者比率・看取り期件数・医療依存度件数・退院退所件数・緊急時対応件数等
- 表示しないもの: 個別利用者の氏名・被保険者番号・住所・電話・生年月日・家族情報・医療機関名・具体的病名

**注意**: user_summary は「要件確認補助」であり、算定可否を保証するものではありません。

### 6. 不足証跡チェックリスト

要件ロジック評価で不足している証跡を確認作業用に整理します。

- 不足証跡 / 推奨確認資料 / 優先度（高・中・低）
- 「any 配下の代替ルートで救われた条件は不足扱いしない」安全弁付き
- `not_applicable` 加算はチェックリストから除外

### 7. 次アクション提示

各不足証跡に「次にやるべき確認行動」を併記します。

例:
- 「月次会議の開催実績と議事録を整理する」
- 「対象期間内の重度者割合の集計根拠を確認する」
- 「ヘルパー在籍者の介護福祉士比率を確認する」

ただし、これらは**確認作業の方向性**を示すものであり、算定可否を助言するものではありません。

## レポートに出る情報

レポート（Markdown）には以下が含まれます。

1. PDF取込結果サマリ（推定利用者数・要介護度分布・抽出信頼度）
2. PDFで算定中として検出された加算
3. 結論サマリ（取得済 / 確認待ち / 対象外 / 情報不足）
4. すぐ確認すべき項目TOP5
5. 取得可能性が高い加算（単位・要件・増収目安）
6. 対象外/取得不可の加算（地域要件等で対象外の理由）
7. 確認待ち項目
8. 情報不足の5分類
9. 必要書類チェックリスト
10. 追加確認すべき職員/利用者情報
11. 増収見込み（40名想定の年間目安）
12. 根拠マスタのバージョン
13. 要件ロジック評価（alpha）
14. 利用者データ連携（DEMO alpha・集計値のみ）alpha.5.4 追加
15. 職員データ連携（DEMO alpha・集計値のみ）
16. 不足証跡チェックリスト（推奨確認資料・優先度・次アクション）

## ただし算定可否保証ではない

本ツールは「公式根拠確認済みの要件に対する機械的な充足推定」を提示するMVPです。

- 算定可否の法的保証はしません
- 最終確認は事業所資料・届出状況・自治体確認が必要です
- 帳票形式や自治体運用差により判定が変動する可能性があります
- マスタの検証ステータス（unverified / checked / legally_reviewed）を確認の上、本番算定判断には `checked` 以上を採用してください

## 導入時に必要な資料

| 資料 | 必須/推奨 | 用途 |
|---|---|---|
| 事業所コード | 必須 | サービス種別・地域単価の決定 |
| 介護給付費明細書PDF | 推奨 | 現状算定加算の自動抽出 |
| 職員資格・配置情報 | 推奨 | 配置要件の機械的充足推定 |
| 利用者構成集計（要介護度・認知症自立度等） | 推奨 | 利用者割合要件の機械的充足推定 |
| 確認待ち項目の整備状況 | 任意 | レポート精度の向上 |

最初は**事業所コード + サービス種別**だけでも判定可能です。データを追加するほど unknown が減り、判定精度が向上します。

---

## 注意点

- 加算算定可否を**法的に保証するものではありません**
- 帳票形式（介護ソフトベンダー差異・自治体差異）により抽出精度が変動します
- 公式サービスコード表との完全照合は継続更新対象です
- 最終確認は事業所資料・届出状況・自治体確認が必要です
- 訪問看護の介護保険版と医療保険版は制度体系が異なります
