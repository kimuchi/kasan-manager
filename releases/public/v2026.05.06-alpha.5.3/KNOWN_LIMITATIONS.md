# 既知の制限事項 v2026.05.06-alpha.5.3

本リリースは**限定外販MVP / 公開デモ版**です。以下の制限を踏まえてご利用ください。

---

## 1. 算定可否の保証はしない

- 本ツールは**加算算定可否を法的に保証するものではありません**
- 「公式根拠確認済みの要件に対する機械的な充足推定」を提示するツールです
- 最終的な届出・算定判断は各自治体の指導課・監査担当および顧問の社労士等にご確認ください

## 2. PDF取込の制約

- **PDF検出は算定中の推定**です。要件充足を保証するものではありません
- **PDF未検出は未算定を意味しません**。サービスコード未収載・帳票形式の差異・抽出ロジック未対応の可能性があります
- **帳票形式（介護ソフトベンダー差異・自治体差異）により抽出精度が変動**します
- 本番運用前に各事業所のレセプトで検証することを推奨します

## 3. サービスコード照合の継続更新

- `service_code_mapping_status` は一部 `pattern_based_unverified`（暫定パターンによる推定）が残ります
- **公式サービスコード表との完全照合は継続更新対象**です
- サービスコード依存の条件は `blocked_by_unverified_mapping` で保留扱いになる場合があります

## 4. 公開デモ用データの位置付け

- `staff.json` は**公開デモ用の架空職員サマリ**です。本番tenant向けのスキーマは確定していません
- `tenant_status` は**公開デモ用の架空証跡データ**です。本番tenant向けの収集フォーマットは確定していません
- 本番運用前にスキーマ凍結・PII方針の最終整備が必要です

## 5. 利用者個別データ未連携

- **`user_summary` / `user.json` は未実装**です（次バージョン以降の予定）
- 重度者割合・看取り期実績件数等の利用者属性 facts は、現在 `tenant_status` 経由でのみ供給されます

## 6. サービス対応範囲

- **訪問看護（医療保険）は別管理**です。介護保険版とは制度体系が異なるため混同しないでください
- **小規模多機能型居宅介護は未 implemented** です（次以降）
- **特別養護老人ホームは planned**（マスタ未着手）です
- **障害福祉サービス**（居宅介護・就労継続支援A/B）は draft（マスタ実装中）です

## 7. 自治体・保険者判断

- 加算算定の最終可否は**自治体・保険者の運用差**が影響します
- 本ツールは法令告示・通知ベースで機械評価しますが、自治体ローカルルールは別途確認が必要です
- 特に**居宅介護支援(I) の要介護3以上40%要件**は地域包括支援センター紹介ケース除外が必要で、PDFだけでは確定しません

## 8. 評価対象外

- `source_status != checked` の要件は評価せず `not_evaluated_source_required` を返します
- `logic_status != checked` のロジックは評価せず `not_evaluated_logic_unchecked` を返します
- `applicability == not_applicable` の加算は評価せず `not_applicable` を返します（改善候補・収益機会には含めません）

## 設計上の安全弁

| 状態 | 表示 | 意味 |
|---|---|---|
| `clear` | ✅ 達成 | 公式根拠確認済み要件が機械的に充足 |
| `not_clear` | ❌ 未達 | 評価条件が満たされていない |
| `partially_clear` | 🟡 部分達成 | 一部条件は達成・一部は不足 |
| `blocked_by_missing_evidence` | 📭 不足証跡あり | 確認資料・データの追加が必要 |
| `blocked_by_unverified_mapping` | 🔒 保留 | サービスコード照合未完了のため評価保留 |
| `not_evaluated_source_required` | ⏳ 根拠未確認 | 公式根拠の確認待ち |
| `not_evaluated_logic_unchecked` | ⏳ ロジック未確認 | 要件論理式の社内レビュー前 |
| `not_applicable` | 🚫 当サービス対象外 | 別サービス専用加算等 |

## マスタの検証ステータス

| ステータス | 意味 |
|---|---|
| `unverified` | 取り込み済み・内部レビュー前（参考表示） |
| `checked` | 社内レビュー完了 |
| `legally_reviewed` | 法務レビュー完了（本番判定の正本） |

`unverified` の段階での判定は参考扱いとし、本番算定判断には `checked` 以上を採用してください。
