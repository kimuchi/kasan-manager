個人情報・公開データ方針 v2026.05.06-alpha.5.3

公開デモ版で外部に出る情報の安全方針です。本リリースパックの全ドキュメントとサンプル出力に適用します。

---

## 1. 公開サンプルは架空データのみ

- 公開サンプルはすべて公開デモ用の架空サンプルです
- 実利用者・実職員・実事業所の情報は含まれません
- 事業所コードは DEMO-0004 / DEMO-0005 / DEMO-0006 / DEMO-0007 の架空コードを使用します

## 2. 公開出力に含めない情報

| 項目 | 公開出力 |
|---|---|
| 実利用者氏名 | 含めない |
| 被保険者番号（10桁） | 含めない |
| 実利用者の住所・電話・生年月日 | 含めない |
| 実職員氏名 | 含めない |
| 実職員の住所・電話・生年月日 | 含めない |
| 実職員の給与額・賞与額 | 含めない |
| 資格証番号・免許番号 | 含めない |
| 入職日・退職日 | 含めない |
| 実事業所コード（10桁の指定事業所番号） | 含めない |
| 実社名（事業者名・施設名） | 含めない |
| 家族情報・医療機関名・具体的病名 | 含めない |

## 3. 職員サマリ表示の方針

公開出力では集計値のみを表示します。

- 表示する: 件数（看護職員数・介護職員数等）・常勤換算合計・比率（介護福祉士比率等）
- 表示しない: staff_id・display_label・個別の資格詳細・個人別シフト

職員サマリは公開デモ用の架空職員サマリから生成され、本番tenantのデータとは無関係です。

## 4. 利用者集計表示の方針（alpha.5.4 追加）

公開出力では集計値のみを表示します。

- 表示する: 利用者総数・要介護度分布（分類ごとの件数）・要介護3以上比率・重度者比率・看取り期関連件数・医療依存度件数・認知症関連件数・退院退所支援関連件数・緊急時対応関連件数
- 表示しない: 個別利用者の氏名・被保険者番号・住所・電話・生年月日・家族情報・医療機関名・具体的病名

利用者集計は公開デモ用の架空集計から生成され、本番tenantのデータとは無関係です。

## 5. raw データの外部提出禁止

| ファイル種別 | 外部提出 |
|---|---|
| 公開デモ用の架空職員サマリ（生ファイル） | 提出しない |
| 公開デモ用の架空証跡データ（生ファイル） | 提出しない |
| 公開デモ用の架空利用者集計（生ファイル） | 提出しない |
| evidence JSON（PDF抽出結果） | 提出しない |
| 内部レポート（DEMO接尾辞なし） | 提出しない |
| 加算マスタファイル本体 | 提出しない |
| 内部資料・社内regulatory資料への参照 | 提出しない |

公開してよいのは以下のみです。

- 公開サンプルレポート
- 公開リリースノート
- 公開README
- 本リリースパック配下

## 6. 内部表現の publicへの非露出

- 内部資料パス（社内マスタ参照等）は public に出さない
- 内部用語（架空データ用の社内呼称等）は public に出さない
- 内部資料への参照（社内regulatory資料等）を public に書かない
- 代わりに「公開デモ用の架空サンプル」「公開デモ用の架空職員サマリ」「公開デモ用の架空証跡データ」「公開デモ用の架空利用者集計」と表現する

## 7. 危険表現の禁止

公開出力では以下のような断定的・誤解を招く表現を使わない:

- 算定可否を法的に保証するかのような断定文言
- 算定が確実視される文言
- PDF未検出を未算定と断定する文言
- 自治体確認の必要性を否定する文言
- 「100%」「絶対」など完全保証を意味する文言

否定形で使うのは可:

- 「算定可否を保証するものではありません」
- 「未検出は未算定を意味しません」
- 「対象外加算が改善候補・収益機会に混入しない」

## 8. public出力前のチェック

公開出力を生成・更新する前に必ず以下を通します。

- 禁止語チェック（実社名・実事業所コード・実職員名・内部表現）
- PIIチェック（被保険者番号・電話・メール・生年月日・給与・住所）
- 危険表現チェック（保証・必ず・絶対・100%）
- raw staff露出チェック（staff_id の値露出・個別職員リスト）
- raw tenant_status露出チェック（個別fact値の生露出）
- raw user露出チェック（個別利用者ID・名前・住所等）
- not_applicable 誤表示チェック（対象外加算が改善候補・収益機会に混ざっていない）
- pattern_based_unverified の誤 clear チェック（サービスコード照合未完了fact が clear になっていない）

すべて pass してから公開してください。
