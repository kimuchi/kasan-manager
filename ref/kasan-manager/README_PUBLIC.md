## CareLinker 加算チェッカーとは

介護事業所が日常的に取り扱う加算は、介護保険だけで100種類を超え、要件は法令告示・通知・Q&A・訂正通知で頻繁に変化します。現場では以下のような「取り漏れ」が日常的に発生しています：

- 「口腔機能向上加算には歯科衛生士が必要」と思い込んでいる（→ 看護職員でも可）
- 「機能訓練指導員はPTでなければならない」と思い込んでいる（→ 看護師・柔整師・あマ指師でも可）
- 「管理栄養士は常勤配置必須」と思い込んでいる（→ 外部連携で可）
- 「サービス提供体制強化加算Ⅰは介護福祉士70%必須」と思い込んでいる（→ 勤続10年以上介護福祉士25%でも可）

**CareLinker 加算チェッカーは、こうした「取り漏れ」を機械的に検出し、取得に必要なアクションを提示するツールです。**

## 誰向けか

- 介護事業所（訪問介護・居宅介護支援・通所介護・訪問看護・小規模多機能・特養 等）の経営者・管理者
- 多施設展開している介護法人の本部・経理担当
- 障害福祉サービス事業者（就労継続支援A/B・居宅介護 等）
- 介護コンサルタント・社労士・行政書士

## 何が分かるか

| 出力項目 | 内容 |
|---|---|
| 結論サマリ | 全加算中、取得可能性が高い加算は何件か |
| すぐ確認すべき項目TOP5 | 行動につながる優先度付きリスト |
| 取得可能性が高い加算 | 単位・要件・増収目安 |
| 対象外/取得不可の加算 | 地域要件等で対象外の理由 |
| 確認待ち項目 | 現場確認・書類整備の進捗 |
| ❔ 情報不足の5分類 | 何があれば判定できるか |
| 必要書類チェックリスト | 加算ごとの届出・運用書類 |
| 追加確認すべき職員/利用者情報 | 判定精度を上げる入力項目 |
| 増収見込み | 40名想定の年間目安 |
| 根拠マスタのバージョン | 法令出典・改定タグ・適用開始日 |

## 介護ソフトを置き換えないこと

CareLinker 加算チェッカーは、**既存の介護ソフト（カイポケ・ほのぼの・ワイズマン・トリケアトプス等）を置き換える製品ではありません**。

- レセプト作成・国保連請求は引き続き既存ソフトで行ってください
- 本ツールは既存ソフトの**上位レイヤー**として、加算最適化に特化します
- レセプトPDFを取り込んで現状算定加算を抽出する機能を提供予定（次工程）

### 要件ロジック評価（alpha・v2026.05.06-alpha.5.3 で導入）

- **DEMO 職員データブリッジ追加（v2026.05.06-alpha.5.3）** — DEMO用の架空 `staff.json` を `--staff-data` オプションで取り込み、`staff_summary.*` factsとして要件DSL評価に供給
- **集計値のみ表示** — レポートには件数・常勤換算合計・比率のみ表示。個別の氏名・staff_id・資格詳細は表示しません
- **本番運用専用のスキーマ凍結は未着手** — α5.3 は公開デモ用の架空サンプルにのみ対応。本番運用前にスキーマ凍結とPII方針の最終整備が必要

- **公式根拠確認済みの要件のみ機械的に評価** — JSON-DSLベースの AND / OR / ネスト条件評価
- **PDF evidenceに基づく機械的推定** — 帳票形式により抽出精度が変動します
- **算定可否を法的に保証するものではありません** — 算定可否の最終確認は事業所資料・届出状況・自治体確認が必要
- **不足証跡がある場合は事業所資料・職員情報・利用者状態等の確認が必要** — `blocked_by_missing_evidence` で表示
- **公式根拠未確認の要件は評価しない** — `not_evaluated_source_required` / `not_evaluated_logic_unchecked` の安全弁
- **当サービス対象外の加算は改善候補・収益機会から除外** — 認知症専門ケア加算(訪問看護では対象外)等

- **対応サービス（PDF取込）**:
    - 通所介護: ✅ 対応済（alpha.4.1）
    - 訪問介護: ✅ 対応済（alpha.4.2・特定事業所加算Ⅰ-Ⅴ含む）
    - 居宅介護支援: ✅ 対応済（alpha.4.3・特定事業所加算Ⅰ-Ⅳ-A・退院退所・ターミナル等含む）
    - 訪問看護（介護保険）: ✅ 対応済（alpha.4.4・緊急時Ⅰ/Ⅱ・特別管理Ⅰ/Ⅱ・ターミナル・看護体制強化等）
    - 訪問看護（医療保険）: 準備中（介護保険版とは制度が異なるため別管理）

- **サービスコード抽出の根拠**:
    - 現バージョンは**暫定パターンによる推定**で実装されています
    - 公式サービスコード表との完全照合は継続更新対象です
    - 帳票形式（介護ソフトベンダー差異・自治体差異）により抽出精度が変動します
    - 本番運用前に各事業所のレセプトで検証することを推奨します

- **居宅介護支援40%要件はPDFだけでは確定しない**:
    - 特定事業所加算(I)の「要介護3以上40%要件」は地域包括支援センター紹介ケース除外が必要
    - PDFから算出した割合は `raw_yokaigo_3plus_ratio` として参考値で保持し、要件clearには使用しません
    - 実際の判定には地域包括紹介ケース台帳との突合が別途必要

> **抽出エンジン版とレポート版の関係**: evidence JSON の `extraction_version` は抽出エンジンのバージョン（v2026.05.06-alpha.5.3）。Markdownレポート末尾の署名はjudge_kasan.pyのレンダラ版（v2026.05.06-alpha.4.1〜）。両者は通常一致しますが、エンジン側のみアップデート時は片方だけ進むことがあります。

- **PDFから算定中加算を推定抽出できる**:
    - サービスコード（通所介護Ⅰ/Ⅱ × 時間帯 × 要介護度）
    - 算定中加算名と件数（推定）
    - 要介護度分布・要介護3以上割合
- **PDF検出は要件充足確認ではない**:
    - PDFで検出された加算は「算定中と推定される」ものであり、配置要件・体制要件・書類整備の充足を意味しない
    - 要件マスタとの整合確認は別途必要
- **PDF未検出は未算定を意味しない**:
    - サービスコード未収載・帳票形式違い・PDF抽出ロジック未対応の可能性がある
    - 「未検出 = 未算定」と判断しない
- **個人情報は保存しない**:
    - 被保険者番号・氏名・住所・電話番号は意図的に抽出・保存しない
    - evidence JSONには集計値・統計値・サービスコードのみを記録
- **実務上の注意**:
    - 帳票形式（ベンダー・自治体差異）により抽出精度が変わるため、本番運用前に各事業所のレセプトで検証が必要

## 注意事項

- **算定可否を法的に保証するものではありません。**
